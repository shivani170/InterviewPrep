    A. What is Lexical Scope?
    - Lexical scope means variables are resolved/accessible based on where functions are defined in the code, not where they are executed.

    = variable’s scope is decided by where it is written in the code (its position), not where it is called from.

B. 2. Closure (the memory capsule 🧪)
A closure is when a function remembers variables from its outer scope even after that outer function has finished execution

- Arrow function takes this from lexical scope, not the object.

- Closures are built on lexical scope
- Closures don’t store values, they store references to variables.
